# CANOPY 植物配對架構

## 目的

植物配對不是性格測驗，而是依使用者可觀察的居家條件，從目前可販售商品中找出相對適合的觀葉植物。結果稱為「條件符合度」，不表示存活率或科學預測機率。

## 資料流

```text
quiz-questions.json
  -> quiz.js 收集與暫存答案
  -> plant-service.js 標準化植物資料
  -> match-rules.json 定義門檻、矩陣與權重
  -> plant-match-service.js 排除與計分
  -> quiz-result.js 渲染商品、原因與注意事項
```

## 必要條件

下列條件不採扣分，而是直接排除：

- 植物尚未發布、不可販售、缺貨或沒有啟用商品規格。
- 寵物可能接觸，但植物資料標示為不適合貓狗接觸。
- 商品超出使用者設定的最高預算。
- 成熟植株所需空間明顯超出可用空間。
- 光照相容度低於規則門檻。
- 擺放位置沒有自然光且使用者不使用植物燈。

## 加權分數

```text
光照 30
濕度 10
照護時間 15
種植經驗 10
空間 15
株型偏好 10
預算 10
```

相容矩陣以 0、0.35、0.7、1 等值表達「不適合、需要調整、可適應、符合」。計分後依 `match-rules.json` 的門檻顯示：

```text
85–100 高度適合
70–84  適合
55–69  有條件適合
54 以下 不列入購買推薦
```

## 安全與資料品質

- 寵物安全資料必須逐筆保留可靠來源參照。
- 植物學名與接受名應保留分類學來源。
- 照護需求應區分理想值、可接受值與應避免條件。
- 若沒有符合安全與商品條件的植物，結果頁必須顯示無結果，不得降低安全門檻。
- 十六筆商品中保留五種標示為對貓狗非毒性的候選植物；寵物可接觸時，其他商品會直接排除。

## 未來 API 邊界

前端目前由 `plant-service.js` 讀取 JSON。未來可將 `load()` 改為呼叫：

```text
GET /api/plants
GET /api/plants/{slug}
```

頁面與配對服務只依賴標準化後的模型，因此不需要跟著更換渲染邏輯。
